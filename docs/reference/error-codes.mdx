# Error Codes

This reference documents all error codes returned by the ItPay Protocol API. Errors are grouped by category. All API errors return a JSON body with the following structure:

```json
{
  "error": {
    "type": "validation_error",
    "code": "invalid_amount",
    "message": "Amount must be greater than 0",
    "param": "amount",
    "status": 400
  }
}
```

## Validation Errors

Errors returned when request data fails validation.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `invalid_amount` | 400 | Amount must be greater than 0 | The provided amount is zero or negative |
| `invalid_amount_format` | 400 | Amount must be an integer representing the smallest currency unit | Amount contains decimals or is not a valid integer |
| `invalid_currency` | 400 | Unsupported or invalid currency code | Currency is not a valid ISO 4217 code or is not supported by the channel |
| `invalid_email` | 400 | Invalid email address format | The provided email address is not valid |
| `invalid_phone` | 400 | Invalid phone number format | The provided phone number is not valid |
| `invalid_url` | 400 | URL must be a valid HTTPS endpoint | The provided URL is not valid or does not use HTTPS |
| `invalid_interval` | 400 | Invalid billing interval. Must be one of: one_time, daily, weekly, monthly, yearly | The interval value is not recognized |
| `invalid_limit` | 400 | Limit must be between 1 and 100 | The pagination limit is out of range |
| `invalid_status` | 400 | Invalid status filter value | The provided status value is not valid for this resource |
| `invalid_metadata` | 400 | Metadata must contain up to 20 key-value pairs with string keys and values | Metadata exceeds limits or contains non-string values |
| `invalid_trial_days` | 400 | Trial period must be between 0 and 365 | Trial period days is out of the allowed range |
| `invalid_events` | 400 | One or more event types are invalid | An unrecognized event type was provided in the events list |
| `invalid_proration` | 400 | Invalid proration behavior. Must be create_prorations or none | The proration_behavior value is not recognized |
| `missing_field` | 400 | Required field `{field}` is missing | A required request parameter was not provided |
| `name_required` | 400 | Manifest name is required | The name field for a manifest is empty |

## Authentication Errors

Errors related to API key authentication.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `missing_api_key` | 401 | No API key provided. Include Authorization header with Bearer token | Request is missing the Authorization header |
| `invalid_api_key` | 401 | Invalid API key | The provided API key is not recognized |
| `expired_api_key` | 401 | API key has expired. Generate a new key from the dashboard | The API key has passed its expiry date |
| `revoked_api_key` | 401 | API key has been revoked | The API key was revoked by the account owner |
| `insufficient_key_permissions` | 403 | API key does not have permission to perform this action | The API key's scope does not include the requested action |

## Authorization Errors

Errors related to resource access permissions.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `unauthorized_resource` | 403 | You do not have access to this resource | The authenticated account does not own or have access to the requested resource |
| `platform_access_denied` | 403 | This action requires platform-level permissions | The API key is not authorized for platform operations |
| `payee_mismatch` | 403 | The resource does not belong to the specified payee | The payee_id in the request does not match the resource's payee |
| `sandbox_forbidden` | 403 | This operation is not available in sandbox mode | The requested operation is restricted to production environment |

## Resource Errors

Errors related to resource existence and state.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `resource_not_found` | 404 | Resource not found | The requested resource does not exist |
| `payment_intent_not_found` | 404 | No payment intent found with the given ID | The specified payment intent ID does not exist |
| `subscription_not_found` | 404 | No subscription found with the given ID | The specified subscription ID does not exist |
| `refund_not_found` | 404 | No refund found with the given ID | The specified refund ID does not exist |
| `manifest_not_found` | 404 | No manifest found with the given ID | The specified manifest ID does not exist |
| `payee_not_found` | 404 | No payee found with the given ID | The specified payee ID does not exist |
| `webhook_endpoint_not_found` | 404 | No webhook endpoint found with the given ID | The specified webhook endpoint ID does not exist |
| `resource_already_exists` | 409 | A resource with this identifier already exists | Duplicate resource creation attempt detected via idempotency key |

## Payment Errors

Errors that occur during payment processing.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `payment_failed` | 402 | Payment processing failed | The payment could not be processed (generic) |
| `insufficient_funds` | 402 | Insufficient funds | The customer's payment method has insufficient funds |
| `card_declined` | 402 | Card was declined | The card issuer declined the transaction |
| `card_expired` | 402 | Card has expired | The provided card is past its expiration date |
| `card_cvv_invalid` | 402 | Invalid CVV code | The card verification code is incorrect |
| `card_velocity_exceeded` | 402 | Card velocity limit exceeded | Too many attempts with this card in a short period |
| `payment_method_invalid` | 400 | Invalid or expired payment method | The payment method is not valid for this transaction |
| `payment_intent_not_succeeded` | 409 | Payment intent must be in succeeded status | Attempting to refund or capture a non-succeeded intent |
| `already_confirmed` | 400 | Payment intent is already confirmed | The payment intent has already been confirmed |
| `cannot_cancel` | 400 | Payment intent cannot be canceled in its current status | The intent status does not allow cancellation |
| `cannot_capture` | 400 | Payment intent is not in requires_capture status | Capture is only allowed for authorized intents |
| `capture_amount_exceeded` | 400 | Capture amount exceeds the authorized amount | The requested capture amount is higher than the authorized amount |
| `exceeds_balance` | 400 | Refund amount exceeds the available balance on the payment intent | The total refunded amount would exceed the original payment amount |
| `already_refunded` | 400 | The full amount of this payment intent has already been refunded | No remaining balance to refund |

## Rate Limit Errors

Errors returned when API rate limits are exceeded.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `rate_limit_exceeded` | 429 | Too many requests. Please retry after `{retry_after}` seconds | The API rate limit has been exceeded |
| `rate_limit_concurrent` | 429 | Too many concurrent requests. Please reduce parallelism | The concurrent request limit has been exceeded |
| `rate_limit_burst` | 429 | Request burst detected. Please slow down | A burst of requests exceeding the limit was detected |

Rate limits are applied on a per-API-key basis:

| Tier | Requests per second | Concurrent requests |
|------|---------------------|---------------------|
| Free | 10 | 2 |
| Pro | 100 | 10 |
| Enterprise | 1000 | 50 |

## Webhook Errors

Errors related to webhook configuration and delivery.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `url_already_exists` | 400 | This URL is already registered as a webhook endpoint | Duplicate webhook URL registration |
| `webhook_delivery_failed` | 500 | Failed to deliver webhook to endpoint | The webhook endpoint did not respond successfully |
| `webhook_secret_rotation_failed` | 500 | Failed to rotate webhook secret | An error occurred while generating a new webhook secret |
| `webhook_endpoint_disabled` | 200 | Webhook endpoint has been disabled due to repeated delivery failures | Automatic disablement after exceeding retry threshold |

## Channel Errors

Errors related to payment channel adapters.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `channel_not_found` | 404 | No channel adapter found with the name '`{channel_name}`' | The specified channel is not registered |
| `channel_not_initialized` | 500 | Channel adapter has not been initialized | The channel adapter was not configured before use |
| `channel_unsupported_currency` | 400 | Channel does not support currency '`{currency}`' | The currency is not in the channel's supported list |
| `channel_unsupported_payment_method` | 400 | Channel does not support payment method '`{method}`' | The payment method is not supported by this channel |
| `channel_connection_failed` | 502 | Failed to connect to payment provider | The channel adapter could not reach the payment provider |
| `channel_timeout` | 504 | Payment provider request timed out | The channel adapter request exceeded the timeout |
| `channel_provider_error` | 502 | Payment provider returned an error: `{provider_message}` | The payment provider returned a non-success response |

## Server Errors

Unexpected errors on the ItPay Protocol server.

| Code | HTTP Status | Message Template | Description |
|------|-------------|-------------------|-------------|
| `internal_error` | 500 | An unexpected error occurred. Please try again later | Generic internal server error |
| `service_unavailable` | 503 | Service is temporarily unavailable. Please try again later | The service is down for maintenance or overloaded |
| `database_error` | 500 | A database error occurred | Internal database operation failed |
| `provider_down` | 502 | A payment provider is temporarily unavailable | Underlying payment provider is unreachable |
| `timeout` | 504 | Request timed out | The request took too long to process |

## Error Response Format

All errors follow this standard JSON format:

```json
{
  "error": {
    "type": "validation_error",
    "code": "invalid_amount",
    "message": "Amount must be greater than 0",
    "param": "amount",
    "status": 400,
    "request_id": "req_abc123def456"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | Error category (snake_case) |
| `code` | string | Machine-readable error code |
| `message` | string | Human-readable error description |
| `param` | string | The request parameter that caused the error (if applicable) |
| `status` | integer | HTTP status code |
| `request_id` | string | Unique identifier for the request (for support) |

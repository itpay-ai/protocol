Subscriptions API

The Subscriptions API enables recurring billing. Subscriptions automatically create payment intents at the interval defined in the associated manifest.

POST /v1/subscriptions​

Create a new subscription.

Parameters​

ParameterTypeRequiredDescription
manifest_idstringyesID of the manifest defining the subscription plan
payee_idstringyesID of the payee receiving subscription payments
customer.emailstringyesCustomer email address
customer.namestringnoCustomer full name
customer.phonestringnoCustomer phone number
currencystringyesThree-letter ISO 4217 currency code
trial_period_daysintegernoNumber of days for the trial period (default: 0)
payment_method_idstringnoID of an existing payment method to use
idempotency_keystringnoIdempotency key for safe retries
metadataobjectnoUp to 20 key-value pairs of custom data
descriptionstringnoDescription of the subscription

Returns​

Returns a Subscription object.

Example​

curl -X POST https://api.itpay.io/v1/subscriptions \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: idem_sub_001" \
  -d '{
    "manifest_id": "mf_abc123",
    "payee_id": "pa_xyz456",
    "currency": "USD",
    "customer": {
      "email": "customer@example.com",
      "name": "Jane Doe"
    },
    "trial_period_days": 14,
    "metadata": {
      "referrer": "direct",
      "plan_tier": "premium"
    }
  }'

Response​

{
  "id": "sub_abc123def456",
  "object": "subscription",
  "manifest_id": "mf_abc123",
  "payee_id": "pa_xyz456",
  "customer": {
    "email": "customer@example.com",
    "name": "Jane Doe"
  },
  "currency": "USD",
  "amount": 2999,
  "interval": "monthly",
  "status": "trialing",
  "trial_start": 1717094400,
  "trial_end": 1718304000,
  "current_period_start": 1717094400,
  "current_period_end": 1718304000,
  "cancel_at_period_end": false,
  "metadata": {
    "referrer": "direct",
    "plan_tier": "premium"
  },
  "created": 1717094400,
  "updated": 1717094400
}

Errors​

HTTP StatusError CodeDescription
400invalid_currencyUnsupported or invalid currency code
400missing_manifestManifest ID is required
400invalid_trial_daysTrial period must be between 0 and 365 days
404manifest_not_foundNo manifest found with the given ID
404payee_not_foundNo payee found with the given ID
402initial_payment_failedThe initial subscription payment failed

GET /v1/subscriptions​

List subscriptions with filtering and pagination.

Parameters​

ParameterTypeRequiredDescription
limitintegernoNumber of subscriptions to return (1-100, default: 10)
starting_afterstringnoCursor for pagination — return results after this ID
ending_beforestringnoCursor for pagination — return results before this ID
statusstringnoFilter by status (trialing, active, past_due, canceled, incomplete, expired)
payee_idstringnoFilter by payee ID
manifest_idstringnoFilter by manifest ID
customer.emailstringnoFilter by customer email

Returns​

Returns a list with data, has_more, and pagination cursors.

Example​

curl "https://api.itpay.io/v1/subscriptions?limit=25&status=active" \
  -H "Authorization: Bearer sk_live_itpay_xyz789"

Response​

{
  "object": "list",
  "data": [
    {
      "id": "sub_abc123def456",
      "object": "subscription",
      "customer": { "email": "customer@example.com" },
      "amount": 2999,
      "currency": "USD",
      "interval": "monthly",
      "status": "active",
      "current_period_end": 1719772800,
      "created": 1717094400
    },
    {
      "id": "sub_ghi789jkl012",
      "object": "subscription",
      "customer": { "email": "buyer@example.com" },
      "amount": 999,
      "currency": "USD",
      "interval": "monthly",
      "status": "trialing",
      "trial_end": 1718304000,
      "created": 1717094000
    }
  ],
  "has_more": true,
  "url": "/v1/subscriptions"
}

Errors​

HTTP StatusError CodeDescription
400invalid_statusInvalid subscription status filter

POST /v1/subscriptions/:id/cancel​

Cancel an active subscription.

Parameters​

ParameterTypeRequiredDescription
idstringpathThe ID of the subscription to cancel
cancel_at_period_endbooleannoIf true, the subscription continues until the end of the current period. If false, cancels immediately. Default: true
reasonstringnoReason for cancellation
metadataobjectnoAdditional metadata for the cancellation

Returns​

Returns a Subscription object with status updated.

Example​

curl -X POST https://api.itpay.io/v1/subscriptions/sub_abc123def456/cancel \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -d '{
    "cancel_at_period_end": true,
    "reason": "customer_request"
  }'

Response​

{
  "id": "sub_abc123def456",
  "object": "subscription",
  "status": "active",
  "cancel_at_period_end": true,
  "canceled_at": 1717094600,
  "current_period_end": 1719772800,
  "metadata": {
    "cancellation_reason": "customer_request"
  },
  "created": 1717094400,
  "updated": 1717094600
}

Errors​

HTTP StatusError CodeDescription
400cannot_cancelSubscription cannot be canceled in its current status
404subscription_not_foundNo subscription found with the given ID

POST /v1/subscriptions/:id/update​

Update a subscription's plan, trial, or metadata.

Parameters​

ParameterTypeRequiredDescription
idstringpathThe ID of the subscription to update
manifest_idstringnoNew manifest ID to switch to a different plan
trial_period_daysintegernoExtend trial by the given number of days
metadataobjectnoMetadata to merge (replaces existing keys)
descriptionstringnoNew description
proration_behaviorstringnoHow to handle proration on plan change (create_prorations, none). Default: create_prorations

Returns​

Returns a Subscription object with updated fields.

Example​

curl -X POST https://api.itpay.io/v1/subscriptions/sub_abc123def456/update \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -d '{
    "manifest_id": "mf_def789",
    "metadata": {
      "plan_tier": "enterprise"
    },
    "proration_behavior": "create_prorations"
  }'

Response​

{
  "id": "sub_abc123def456",
  "object": "subscription",
  "manifest_id": "mf_def789",
  "amount": 9999,
  "currency": "USD",
  "interval": "monthly",
  "status": "active",
  "current_period_start": 1717094400,
  "current_period_end": 1719772800,
  "metadata": {
    "plan_tier": "enterprise"
  },
  "created": 1717094400,
  "updated": 1717094800
}

Errors​

HTTP StatusError CodeDescription
400cannot_updateSubscription cannot be updated in its current status
400invalid_prorationInvalid proration_behavior value
404subscription_not_foundNo subscription found with the given ID
404manifest_not_foundNo manifest found with the given ID

Subscription Statuses​

StatusDescription
trialingTrial period is active, no payment collected yet
activeSubscription is active and payments are being collected
past_dueThe most recent payment failed; retries are in progress
canceledSubscription has been canceled (immediately or at period end)
incompleteInitial payment failed; subscription is incomplete
expiredSubscription has reached its end date or all billing cycles completed

Supported Intervals​

IntervalDescription
dailyBilled every day
weeklyBilled every week
monthlyBilled every month
yearlyBilled every year

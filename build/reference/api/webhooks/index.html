Webhooks API

Webhooks provide real-time notifications about events in your ItPay account. Configure webhook endpoints to receive POST requests when events occur, such as successful payments, failed charges, subscription renewals, and refund completions.

POST /v1/webhook_endpoints​

Create a new webhook endpoint.

Parameters​

ParameterTypeRequiredDescription
urlstringyesHTTPS URL where events will be sent
eventsarray<string>yesList of event types to subscribe to. Use * for all events
descriptionstringnoDescription of this webhook endpoint
metadataobjectnoUp to 20 key-value pairs of custom data
enabledbooleannoWhether the endpoint is active immediately (default: true)
api_versionstringnoAPI version to use when sending events (default: current version)
idempotency_keystringnoIdempotency key for safe retries

Returns​

Returns a WebhookEndpoint object including the signing secret.

Example​

curl -X POST https://api.itpay.io/v1/webhook_endpoints \
  -H "Authorization: Bearer *** \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://api.example.com/webhooks/itpay",
    "events": [
      "payment_intent.succeeded",
      "payment_intent.failed",
      "subscription.renewed",
      "refund.completed"
    ],
    "description": "Production webhook for payment notifications",
    "metadata": {
      "environment": "production",
      "team": "payments"
    }
  }'

Response​

{
  "id": "we_abc123def456",
  "object": "webhook_endpoint",
  "url": "https://api.example.com/webhooks/itpay",
  "events": [
    "payment_intent.succeeded",
    "payment_intent.failed",
    "subscription.renewed",
    "refund.completed"
  ],
  "description": "Production webhook for payment notifications",
  "secret": "whsec_abc123def456ghijklmnopqrstuvwxyz",
  "status": "enabled",
  "metadata": {
    "environment": "production",
    "team": "payments"
  },
  "created": 1717094400,
  "updated": 1717094400
}

Signature Verification​

All webhook requests include an Itpay-Signature header. Verify signatures to ensure requests are legitimate.

Verification Steps​

  1. Extract the signature from the Itpay-Signature header
  2. Get the raw request body (do not parse or modify it)
  3. Compute an HMAC-SHA256 hash of the body using your webhook secret
  4. Compare the computed signature with the header value (use constant-time comparison)

TypeScript Example​

import crypto from 'crypto';

function verifyWebhookSignature(
  rawBody: Buffer,
  signature: string,
  secret: string
): boolean {
  const computed = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');
  const expected = `sha256=${computed}`;
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected)
  );
}

// Usage in Express
app.post('/webhooks/itpay', express.raw({ type: 'application/json' }), (req, res) => {
  const sig = req.headers['itpay-signature'] as string;
  const secret = 'whsec_abc123def456ghijklmnopqrstuvwxyz';

  if (!verifyWebhookSignature(req.body, sig, secret)) {
    return res.status(400).json({ error: 'Invalid signature' });
  }

  const event = JSON.parse(req.body.toString());
  // Process event...
  res.json({ received: true });
});

Go Example​

import (
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "fmt"
    "net/http"
)

func verifyWebhookSignature(payload []byte, signature, secret string) bool {
    mac := hmac.New(sha256.New, []byte(secret))
    mac.Write(payload)
    expected := fmt.Sprintf("sha256=%s", hex.EncodeToString(mac.Sum(nil)))
    return hmac.Equal([]byte(signature), []byte(expected))
}

func webhookHandler(w http.ResponseWriter, r *http.Request) {
    signature := r.Header.Get("Itpay-Signature")
    secret := "whsec_abc123def456ghijklmnopqrstuvwxyz"

    buf := new(bytes.Buffer)
    buf.ReadFrom(r.Body)
    payload := buf.Bytes()

    if !verifyWebhookSignature(payload, signature, secret) {
        http.Error(w, "Invalid signature", http.StatusBadRequest)
        return
    }

    // Process event...
    w.WriteHeader(http.StatusOK)
    json.NewEncoder(w).Encode(map[string]bool{"received": true})
}

Retry Policy​

Webhook delivery follows an exponential backoff retry schedule:

AttemptDelay After Previous Attempt
1stImmediate
2nd10 seconds
3rd1 minute
4th10 minutes
5th1 hour
6th6 hours
7th24 hours
8th+Every 24 hours (up to 7 days)

After 7 days of continuous failures, the endpoint is automatically disabled and a notification is sent to the account email.

Best Practices​

  • Respond with 200 OK as quickly as possible — process events asynchronously
  • Do not parse the body before verifying the signature
  • Store the event id to deduplicate in case of retries
  • Log all received events for debugging
  • Configure alerts for webhook delivery failures
  • Rotate webhook secrets periodically via the dashboard

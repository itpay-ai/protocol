Payments API

The Payments API provides a complete payment intent lifecycle. Use it to create, confirm, capture, cancel, and retrieve one-time and recurring payments.

POST /v1/payment_intents​

Create a new payment intent.

Parameters​

ParameterTypeRequiredDescription
amountintegeryesAmount in smallest currency unit (cents, kobo, etc.)
currencystringyesThree-letter ISO 4217 currency code (e.g., "USD", "NGN")
manifest_idstringyesID of the manifest defining payment rules
payee_idstringyesID of the payee receiving the payment
channelsarray<string>noAllowed payment channels (e.g., ["card", "bank_transfer"])
customer.emailstringyesCustomer email address
customer.namestringnoCustomer full name
customer.phonestringnoCustomer phone number
idempotency_keystringnoIdempotency key for safe retries
descriptionstringnoDescription of the payment
metadataobjectnoUp to 20 key-value pairs of custom data
return_urlstringnoURL to redirect the customer after payment

Returns​

Returns a PaymentIntent object.

Example​

curl -X POST https://api.itpay.io/v1/payment_intents \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: idem_req_001" \
  -d '{
    "amount": 2999,
    "currency": "USD",
    "manifest_id": "mf_abc123",
    "payee_id": "pa_xyz456",
    "channels": ["card", "bank_transfer"],
    "customer": {
      "email": "customer@example.com",
      "name": "Jane Doe"
    },
    "description": "Premium Widget Package - Monthly",
    "metadata": {
      "plan_tier": "premium"
    }
  }'

Response​

{
  "id": "pi_abc123def456",
  "object": "payment_intent",
  "amount": 2999,
  "amount_capturable": 0,
  "amount_received": 0,
  "currency": "USD",
  "status": "requires_payment_method",
  "client_secret": "pi_abc123def456_secret_xyz789",
  "manifest_id": "mf_abc123",
  "payee_id": "pa_xyz456",
  "customer": {
    "email": "customer@example.com",
    "name": "Jane Doe"
  },
  "channels": ["card", "bank_transfer"],
  "description": "Premium Widget Package - Monthly",
  "metadata": {
    "plan_tier": "premium"
  },
  "created": 1717094400,
  "updated": 1717094400
}

Errors​

HTTP StatusError CodeDescription
400invalid_amountAmount must be greater than 0
400invalid_currencyUnsupported or invalid currency code
400missing_manifestManifest ID is required
402payment_failedPayment processing failed
404manifest_not_foundNo manifest found with the given ID
404payee_not_foundNo payee found with the given ID

Retrieve a payment intent by its ID.

ParameterTypeRequiredDescription

GET /v1/payment_intents​

List payment intents with filtering and pagination.

Parameters​

ParameterTypeRequiredDescription
limitintegernoNumber of intents to return (1-100, default: 10)
starting_afterstringnoCursor for pagination — return results after this ID
ending_beforestringnoCursor for pagination — return results before this ID
statusstringnoFilter by status (requires_payment_method, requires_confirmation, requires_capture, succeeded, failed, canceled)
payee_idstringnoFilter by payee ID
manifest_idstringnoFilter by manifest ID
created.gteintegernoUnix timestamp — return intents created at or after
created.lteintegernoUnix timestamp — return intents created at or before

Returns​

Returns a list with data, has_more, and pagination cursors.

Example​

curl "https://api.itpay.io/v1/payment_intents?limit=25&status=succeeded" \
  -H "Authorization: Bearer sk_live_itpay_xyz789"

Response​

{
  "object": "list",
  "data": [
    {
      "id": "pi_abc123def456",
      "object": "payment_intent",
      "amount": 2999,
      "currency": "USD",
      "status": "succeeded",
      "customer": { "email": "customer@example.com" },
      "created": 1717094400
    },
    {
      "id": "pi_ghi789jkl012",
      "object": "payment_intent",
      "amount": 999,
      "currency": "USD",
      "status": "requires_payment_method",
      "customer": { "email": "buyer@example.com" },
      "created": 1717094000
    }
  ],
  "has_more": true,
  "url": "/v1/payment_intents"
}

Errors​

HTTP StatusError CodeDescription
400invalid_limitLimit must be between 1 and 100

POST /v1/payment_intents/:id/confirm​

Confirm a payment intent, triggering the actual payment processing.

Parameters​

ParameterTypeRequiredDescription
idstringpathThe ID of the payment intent to confirm
payment_method_idstringnoID of a saved payment method to use
payment_method_dataobjectnoNew payment method details (type, card number, etc.)
return_urlstringnoURL to redirect after payment completes

Returns​

Returns a PaymentIntent object with the updated status.

Example​

curl -X POST https://api.itpay.io/v1/payment_intents/pi_abc123def456/confirm \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -d '{
    "payment_method_data": {
      "type": "card",
      "card": {
        "number": "4242424242424242",
        "exp_month": 12,
        "exp_year": 2028,
        "cvc": "123"
      }
    }
  }'

Response​

{
  "id": "pi_abc123def456",
  "object": "payment_intent",
  "amount": 2999,
  "status": "succeeded",
  "amount_received": 2999,
  "payment_method": {
    "type": "card",
    "card": {
      "brand": "visa",
      "last4": "4242"
    }
  },
  "created": 1717094400,
  "updated": 1717094500
}

Errors​

HTTP StatusError CodeDescription
400already_confirmedPayment intent is already confirmed
400invalid_payment_methodThe provided payment method is invalid
402payment_failedPayment processing failed
404payment_intent_not_foundNo payment intent found with the given ID

POST /v1/payment_intents/:id/cancel​

Cancel a payment intent before it is captured.

Parameters​

ParameterTypeRequiredDescription
idstringpathThe ID of the payment intent to cancel
reasonstringnoReason for cancellation (duplicate, fraudulent, requested_by_customer, abandoned)

Returns​

Returns a PaymentIntent object with status set to canceled.

Example​

curl -X POST https://api.itpay.io/v1/payment_intents/pi_abc123def456/cancel \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "requested_by_customer"
  }'

Response​

{
  "id": "pi_abc123def456",
  "object": "payment_intent",
  "amount": 2999,
  "status": "canceled",
  "cancellation_reason": "requested_by_customer",
  "created": 1717094400,
  "updated": 1717094600
}

Errors​

HTTP StatusError CodeDescription
400cannot_cancelPayment intent cannot be canceled in its current status
404payment_intent_not_foundNo payment intent found with the given ID

POST /v1/payment_intents/:id/capture​

Capture funds from an authorized payment intent. Only applicable for intents with requires_capture status.

Parameters​

ParameterTypeRequiredDescription
idstringpathThe ID of the payment intent to capture
amount_to_captureintegernoAmount to capture (must be <= authorized amount). Defaults to full amount.

Returns​

Returns a PaymentIntent object with status set to succeeded.

Example​

curl -X POST https://api.itpay.io/v1/payment_intents/pi_abc123def456/capture \
  -H "Authorization: Bearer sk_live_itpay_xyz789" \
  -H "Content-Type: application/json" \
  -d '{
    "amount_to_capture": 2999
  }'

Response​

{
  "id": "pi_abc123def456",
  "object": "payment_intent",
  "amount": 2999,
  "amount_capturable": 0,
  "amount_received": 2999,
  "status": "succeeded",
  "created": 1717094400,
  "updated": 1717094700
}

Errors​

HTTP StatusError CodeDescription
400cannot_capturePayment intent is not in requires_capture status
400capture_amount_exceededcapture amount exceeds the authorized amount
404payment_intent_not_foundNo payment intent found with the given ID

Payment Intent Statuses​

StatusDescription
requires_payment_methodIntent created but no payment method attached
requires_confirmationPayment method attached, needs confirmation
requires_captureAuthorized but funds not yet captured
processingPayment is being processed
succeededPayment completed successfully
failedPayment processing failed
canceledPayment was canceled before completion

Manifests API

A Manifest defines what products or services a payee offers, including pricing, billing intervals, and metadata. Manifests are created in draft mode and must be published before they can be used to create payment intents or subscriptions.

POST /v1/manifests​

Create a new manifest.

Parameters​

ParameterTypeRequiredDescription
payee_idstringyesID of the payee who owns this manifest
namestringyesHuman-readable name of the product or plan
descriptionstringnoDetailed description of what this manifest offers
amountintegeryesAmount in smallest currency unit (cents, kobo, etc.)
currencystringyesThree-letter ISO 4217 currency code
intervalstringnoBilling interval (one_time, daily, weekly, monthly, yearly). Default: one_time
interval_countintegernoNumber of intervals between billings (default: 1). For example, 3 with interval monthly = every 3 months
trial_period_daysintegernoDefault trial period in days for subscriptions (0-365)
metadataobjectnoUp to 20 key-value pairs of custom data
idempotency_keystringnoIdempotency key for safe retries

Returns​

Returns a Manifest object with status draft.

Example​

curl -X POST https://api.itpay.io/v1/manifests \
  -H "Authorization: Bearer *** \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: idem_mf_001" \
  -d '{
    "payee_id": "pa_xyz456",
    "name": "Premium Widget Package",
    "description": "Access to premium widget features including analytics and priority support",
    "amount": 2999,
    "currency": "USD",
    "interval": "monthly",
    "interval_count": 1,
    "trial_period_days": 14,
    "metadata": {
      "plan_tier": "premium",
      "max_users": "10",
      "features": "analytics,support,api_access"
    }
  }'

Response​

{
  "id": "mf_abc123def456",
  "object": "manifest",
  "payee_id": "pa_xyz456",
  "name": "Premium Widget Package",
  "description": "Access to premium widget features including analytics and priority support",
  "amount": 2999,
  "currency": "USD",
  "interval": "monthly",
  "interval_count": 1,
  "trial_period_days": 14,
  "status": "draft",
  "metadata": {
    "plan_tier": "premium",
    "max_users": "10",
    "features": "analytics,support,api_access"
  },
  "created": 1717094400,
  "updated": 1717094400
}

Errors​

HTTP StatusError CodeDescription
400invalid_amountAmount must be greater than 0
400invalid_currencyUnsupported or invalid currency code
400invalid_intervalInvalid billing interval value
400invalid_trial_daysTrial period must be between 0 and 365
400name_requiredManifest name is required
404payee_not_foundNo payee found with the given ID

List manifests with filtering and pagination.

ending_beforestringnoCursor for pagination — return results before this ID

POST /v1/manifests/:id/deactivate​

Deactivate a published manifest, preventing new payment intents and subscriptions from using it. Existing subscriptions continue until their current period ends.

Parameters​

ParameterTypeRequiredDescription
idstringpathThe ID of the manifest to deactivate
reasonstringnoReason for deactivation

Returns​

Returns a Manifest object with status set to deactivated.

Example​

curl -X POST https://api.itpay.io/v1/manifests/mf_abc123def456/deactivate \
  -H "Authorization: Bearer *** \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "plan_discontinued"
  }'

Response​

{
  "id": "mf_abc123def456",
  "object": "manifest",
  "name": "Premium Widget Package",
  "amount": 2999,
  "currency": "USD",
  "interval": "monthly",
  "status": "deactivated",
  "active": false,
  "deactivated_at": 1717094800,
  "deactivation_reason": "plan_discontinued",
  "created": 1717094400,
  "updated": 1717094800
}

Errors​

HTTP StatusError CodeDescription
400already_deactivatedManifest is already deactivated
400cannot_deactivateManifest cannot be deactivated in its current status
404manifest_not_foundNo manifest found with the given ID

Manifest Statuses​

StatusDescription
draftManifest is being configured but is not yet available for payments
publishedManifest is active and can be used for payment intents and subscriptions
deactivatedManifest is no longer accepting new payments; existing subscriptions continue

Manifest Lifecycle​

  1. Draft — Create the manifest with all configuration options
  2. Publish — Activate the manifest for use in payments and subscriptions
  3. Active use — Payment intents and subscriptions reference the published manifest
  4. Deactivate — Stop new transactions while honoring existing subscriptions
  5. Re-publish — A deactivated manifest can be published again if needed

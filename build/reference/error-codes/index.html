Error Codes

This reference documents all error codes returned by the ItPay Protocol API. Errors are grouped by category. All API errors return a JSON body with the following structure:

{
  "error": {
    "type": "validation_error",
    "code": "invalid_amount",
    "message": "Amount must be greater than 0",
    "param": "amount",
    "status": 400
  }
}

Validation Errors​

Errors returned when request data fails validation.

CodeHTTP StatusMessage TemplateDescription
invalid_amount400Amount must be greater than 0The provided amount is zero or negative
invalid_amount_format400Amount must be an integer representing the smallest currency unitAmount contains decimals or is not a valid integer
invalid_currency400Unsupported or invalid currency codeCurrency is not a valid ISO 4217 code or is not supported by the channel
invalid_email400Invalid email address formatThe provided email address is not valid
invalid_phone400Invalid phone number formatThe provided phone number is not valid
invalid_url400URL must be a valid HTTPS endpointThe provided URL is not valid or does not use HTTPS
invalid_interval400Invalid billing interval. Must be one of: one_time, daily, weekly, monthly, yearlyThe interval value is not recognized
invalid_limit400Limit must be between 1 and 100The pagination limit is out of range
invalid_status400Invalid status filter valueThe provided status value is not valid for this resource
invalid_metadata400Metadata must contain up to 20 key-value pairs with string keys and valuesMetadata exceeds limits or contains non-string values
invalid_trial_days400Trial period must be between 0 and 365Trial period days is out of the allowed range
invalid_events400One or more event types are invalidAn unrecognized event type was provided in the events list
invalid_proration400Invalid proration behavior. Must be create_prorations or noneThe proration_behavior value is not recognized
missing_field400Required field {field} is missingA required request parameter was not provided
name_required400Manifest name is requiredThe name field for a manifest is empty

Authentication Errors​

Errors related to API key authentication.

CodeHTTP StatusMessage TemplateDescription
missing_api_key401No API key provided. Include Authorization header with Bearer tokenRequest is missing the Authorization header
invalid_api_key401Invalid API keyThe provided API key is not recognized
expired_api_key401API key has expired. Generate a new key from the dashboardThe API key has passed its expiry date
revoked_api_key401API key has been revokedThe API key was revoked by the account owner
insufficient_key_permissions403API key does not have permission to perform this actionThe API key's scope does not include the requested action

Authorization Errors​

Errors related to resource access permissions.

CodeHTTP StatusMessage TemplateDescription
unauthorized_resource403You do not have access to this resourceThe authenticated account does not own or have access to the requested resource
platform_access_denied403This action requires platform-level permissionsThe API key is not authorized for platform operations
payee_mismatch403The resource does not belong to the specified payeeThe payee_id in the request does not match the resource's payee
sandbox_forbidden403This operation is not available in sandbox modeThe requested operation is restricted to production environment

subscription_not_found404No subscription found with the given IDThe specified subscription ID does not exist
refund_not_found404No refund found with the given IDThe specified refund ID does not exist
manifest_not_found404No manifest found with the given IDThe specified manifest ID does not exist
payee_not_found404No payee found with the given IDThe specified payee ID does not exist
webhook_endpoint_not_found404No webhook endpoint found with the given IDThe specified webhook endpoint ID does not exist
resource_already_exists409A resource with this identifier already existsDuplicate resource creation attempt detected via idempotency key

Errors that occur during payment processing.

CodeHTTP StatusMessage TemplateDescription
payment_failed402Payment processing failedThe payment could not be processed (generic)
insufficient_funds402Insufficient fundsThe customer's payment method has insufficient funds
card_declined402Card was declinedThe card issuer declined the transaction
card_expired402Card has expiredThe provided card is past its expiration date
card_cvv_invalid402Invalid CVV codeThe card verification code is incorrect
card_velocity_exceeded402Card velocity limit exceededToo many attempts with this card in a short period
payment_method_invalid400Invalid or expired payment methodThe payment method is not valid for this transaction
payment_intent_not_succeeded409Payment intent must be in succeeded statusAttempting to refund or capture a non-succeeded intent
already_confirmed400Payment intent is already confirmedThe payment intent has already been confirmed
cannot_cancel400Payment intent cannot be canceled in its current statusThe intent status does not allow cancellation
cannot_capture400Payment intent is not in requires_capture statusCapture is only allowed for authorized intents
capture_amount_exceeded400Capture amount exceeds the authorized amountThe requested capture amount is higher than the authorized amount
exceeds_balance400Refund amount exceeds the available balance on the payment intentThe total refunded amount would exceed the original payment amount
already_refunded400The full amount of this payment intent has already been refundedNo remaining balance to refund

Rate Limit Errors​

Errors returned when API rate limits are exceeded.

CodeHTTP StatusMessage TemplateDescription
rate_limit_exceeded429Too many requests. Please retry after {retry_after} secondsThe API rate limit has been exceeded
rate_limit_concurrent429Too many concurrent requests. Please reduce parallelismThe concurrent request limit has been exceeded
rate_limit_burst429Request burst detected. Please slow downA burst of requests exceeding the limit was detected

Rate limits are applied on a per-API-key basis:

Pro10010

database_error500A database error occurredInternal database operation failed

timeout504Request timed outThe request took too long to process

Error Response Format​

All errors follow this standard JSON format:

{
  "error": {
    "type": "validation_error",
    "code": "invalid_amount",
    "message": "Amount must be greater than 0",
    "param": "amount",
    "status": 400,
    "request_id": "req_abc123def456"
  }
}
FieldTypeDescription
typestringError category (snake_case)
codestringMachine-readable error code
messagestringHuman-readable error description
paramstringThe request parameter that caused the error (if applicable)
statusintegerHTTP status code
request_idstringUnique identifier for the request (for support)
